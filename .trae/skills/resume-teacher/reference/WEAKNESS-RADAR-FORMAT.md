# weakness-radar.md 格式

**弱点雷达**：一份动态的拷问点（Probe Point）清单，记录每点的当前状态。它是 Interrogator
复访时决定"打哪里"的依据，会被持续修订。

## 结构

```md
# 弱点雷达

> 状态图例：✅ 已攻克 ｜ 🔴 反复翻车（知识盲区）｜ ❓ 存疑待查（经历可能有水分）

| Probe Point | 来源（简历条目） | 状态 | 最近裁定摘要 | 翻车次数 |
|-------------|------------------|------|--------------|----------|
| Redis 缓存穿透处理 | XX项目-性能优化 | 🔴 | 深度弱：只会背概念，讲不出布隆过滤器取舍 | 2 |
| 主导微服务拆分 | YY公司-架构 | ❓ | 真实性存疑：说不清拆分依据与边界 | 1 |
| Go channel 用法 | 技能-Go | ✅ | 正确性/深度均强 | 0 |
```

## 规则

- **每个 Probe Point 一行**，状态随每场盘问更新。
- 🔴 反复翻车 = 知识盲区（深度/正确性反复弱）；❓ 存疑待查 = 真实性存疑（可能吹水）。
- 复访时优先挑 🔴 和 ❓ 的点继续逼问；用户真正补上后改为 ✅。
- "最近裁定摘要"用一句话点出弱在哪个维度，便于快速决策。
- 这是动态文件——**持续修订**，不像 Debrief 那样只增不改。
