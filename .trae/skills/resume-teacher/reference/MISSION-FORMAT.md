# MISSION.md 格式

记录用户的**备战目标（目标岗位画像）**。它是校准一切拷问深度与 Verdict 及格线的标尺。
MISSION 可演进——用户更换目标岗位时更新本文件。

## 结构

```md
# 备战目标

**目标岗位**：{如 后端 / Go、资深前端、全栈、算法}
**级别**：{初级 / 中级 / 资深 / 专家}
**目标公司类型**：{大厂 / 中厂 / 创业公司 / 外企}

## （可选）目标 JD
{若用户贴出招聘 JD，原样粘在此处，拷问时逐条对照}

## 备注
{用户特别在意的方向、想重点被拷问的领域等}
```

## 规则

- 岗位画像三项（岗位 / 级别 / 公司类型）是必填，缺一不可校准。
- JD 可选；有则及格线对照 JD，更精准。
- 保持简短——这是标尺，不是详细需求文档。
