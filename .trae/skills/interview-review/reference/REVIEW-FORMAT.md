# Review 格式

一场**真实面试**复盘后的**独立存档**，记录本场实战答卷的逐题裁定与场级诊断。只增不改，
与模拟场的 Debrief 并列，共同构成用户的成长轨迹。存于 `reviews/`，编号递增：
`0001-<公司-岗位>-review.md`、`0002-...`。

> 与 `debriefs/`（模拟复盘）物理隔离、术语隔离。见根目录 `CONTEXT.md`：**Debrief=模拟场，
> Review=真实面试场**。

## 结构

```md
# 面试复盘 0001：{公司 / 岗位，如 字节-前端}

- **日期**：{YYYY-MM-DD}
- **目标岗位**：{引用 MISSION 的岗位画像，用于解释及格线}
- **文字稿**：{对应 transcripts/000N-... 文件名}

## 逐题裁定（QA Pair）

### QA 1：{面试官问的题，一句话概括}
- **命中类型**：雷达已知点 / 简历相关新点 / 简历外考点
- **五要素覆盖**（仅经历/项目类）：做了什么 ✓/✗ ｜ 初衷目标 _ ｜ 职责边界 _ ｜ 解决了什么 _ ｜ 数据效果 _
- **四维裁定**：正确性 强/中/弱 ｜ 深度 _ ｜ 真实性 强/中/弱/N-A ｜ 表达 _
- **点评 / 纠错 / 补漏**：{结合四维的具体反馈}
- **标准答法**：{这题面试场上本该怎么答，直接可复用}

### QA 2：{标题}
...

## 场级复盘

### 备战验收（雷达已知点对账）
- ✅ 已验收：{🔴/📘 这次真答好了的点}
- 🔴 实战复发：{又塌了的点，翻车次数 +1，补强未进脑}
- 金句未落地：{矫正手册该出现却没说出口的金句}

### 盲区扩面（简历外新考点）
- {本场新暴露、需补进雷达的考点，标来源「真实面试000N」}

### 临场诊断
- **听题准确度**：{有没有听错/答偏}
- **抗压表现**：{被追问时崩没崩、有没有顺着错误前提编}
- **金句到位率**：{备的行话/金句用上了几个}
- **表达流畅度**：{逻辑顺不顺、口语碎片是否影响表达}

## 优化建议（双轨反哺）
- **简历轨**：{实战答塌/答虚的经历，给 resume.md 的降级或补五要素建议}
- **知识轨**：{实战复发的真盲区，建议补进/复习 弱点矫正手册.md 哪一节}

## 下一步
- {建议下轮 resume-teacher 重点重练的点；或简历该改的落点}
```

## 规则

- **只增不改**：每场一份新文件，不回头改旧 Review。
- **四维务必写全**，与弱点雷达的状态回写保持一致；简历外纯知识题的"真实性"维记 **N/A**。
- **场级三项是本格式相对 Debrief 的增量**——备战验收 / 盲区扩面 / 临场诊断，缺一则复盘退化成
  普通逐题点评，失去"真实面试"独有的诊断价值。
- **每题必给标准答法**：复盘的落点是"下次怎么答对"，不是只判死刑。
- **优化建议双轨反哺**：简历轨对齐 Debrief 的口径，知识轨指向矫正手册具体小节，保证闭环。
